# Desafio PicPay Simplificado

Criei este projeto para colocar em prática o que venho aprendendo sobre desenvolvimento de APIs REST com Java e Spring Boot. A ideia foi pegar as regras de negócio do desafio do PicPay (vaga júnior) para consolidar meus estudos.

## O que pratiquei aqui:
* **Estrutura de pastas**: Divisão limpa do projeto usando camadas (Controllers, Services, DTOs e Repositories).
* **Banco de dados relacional**: Mapeamento de entidades com JPA/Hibernate, relacionamentos e travas de campos únicos (como CPF e E-mail no H2).
* **Tratamento de erros**: Criação de um tratador de exceções para capturar erros e devolver respostas organizadas para o cliente.
* **Controle de versão**: Prática de Git e organização do histórico de commits.

## Tecnologias:
* Java
* Spring Boot
* Spring Data JPA
* Banco H2 (In-Memory)
* Postman (para os testes de requisição)
